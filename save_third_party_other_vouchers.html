NutraACS Integration Guide – Save Third Party Other Vouchers API

This API allows third-party providers to push "Other Service" voucher transactions (e.g. Visa, Insurance, Miscellaneous) directly into NutraACS for both Sale and Refund processing.

1. API Endpoint

POST /api/voucher/other/save_third_party_other_vouchers

Content-Type: application/json

2. Request Structure

{
  "strProviderCode": "PROVIDERCODE",
  "objPaymentInfo": { ... },
  "arrVouchers": [
    {
      "strVoucherNo": "OV859360",
      "strTransactionType": "REFUND",
      "strVoucherStatus": "R",
      "strServiceType": "VISA",
      "strRegionCode": "INT",
      "datFrom": "26/01/2024",
      "datTo": "28/01/2024",
      "intNoOfPax": "1",
      "strPaxName": "JITHIN",
      "datRefund": "25/01/2024",
      "strBranch": "MAIN",
      ...
      "objDetails": { ... },
      "objGdsRemarks": { ... },
      "objTransactionalRemarks": { ... }
    }
  ]
}
Field placement note: Unlike hotel and car vouchers, several fields that logically belong to the transaction record are placed at the voucher root level (not inside objDetails): datIssue, datRefund, strIssuingStaffId, strIssuingStaffEmail, strBranch, intNoOfPax, strDimensionCodeOne/Two/Three, objGdsRemarks, and objTransactionalRemarks. objDetails contains only financial data.

3. Top-Level Fields

FieldTypeRequiredDescription
strProviderCode string Required The unique vendor/provider code identifying the third-party system (e.g. "NUFLIGHTS"). Must be pre-registered in NutraACS.
arrVouchers array Required Array of one or more other-service voucher objects to import. Each object represents a single transaction (Sale or Refund). See Section 4.
objPaymentInfo object Optional Payment details for batch invoicing. When provided, all successfully saved vouchers are automatically invoiced. See Section 5.

4. Voucher Object Fields

Each element in arrVouchers represents one other-service transaction. The object is flat — most booking and service metadata lives at the root level, while all financial data goes inside objDetails.

4a. Master / Root Fields

Service type, location, booking, pax, and staff — all at the voucher root level

FieldTypeRequiredDescription & Validation
strVoucherNo string Required Unique other-service voucher number (e.g. "OV859360"). Max 30 characters. Only letters, digits, / and - are allowed — other special characters cause a validation error. Cannot be null or empty; the entire batch is rejected if missing.
strTransactionType string Required Type of transaction. Accepted values: "SALE" or "REFUND" (case-sensitive). Any other value rejects the entire batch.
strVoucherStatus string Required Current status of the voucher. Accepted values:
  • "I" or "ISSUE" — Issued (use with SALE)
  • "R" or "REFUND" — Refunded (use with REFUND)
  • "V" or "VOID" — Voided (use with REFUND)
Status must be consistent with strTransactionType. Mismatch causes a per-voucher error.
strServiceType string Required The other-service category name as configured in NutraACS (e.g. "VISA", "INSURANCE"). Must match an active service name exactly (case-insensitive). If missing, a per-voucher error is raised. If the name does not exist, a separate error is raised.
strRegionCode string Required Tax region code. Accepted values: "INT" (international) or "DOM" (domestic). Required for all sources including THIRD_PARTY. If the provided country is the same as the company's country, strRegionCode must be "DOM"; if different, it must be "INT".
datFrom string (date) Required Service start date (e.g. visa validity start, insurance start). Format: "DD/MM/YYYY". Must not be later than datTo. Must not be earlier than datIssue.
datTo string (date) Required Service end date. Format: "DD/MM/YYYY". Must not be earlier than datFrom. Must not be earlier than datIssue.
datIssue string (date) Cond. Required when strTransactionType is "SALE". The voucher issue date. Format: "DD/MM/YYYY". Must be on or before datFrom and datTo.
datRefund string (date) Cond. Required when strTransactionType is "REFUND". The refund processing date. Format: "DD/MM/YYYY". Must be on or after the original sale voucher's issue date in the database.
intNoOfPax string / number Required Number of passengers/applicants covered by this voucher (e.g. "1"). Must be a positive integer. Must not be zero. Max 5 characters (i.e. ≤ 99999).
strPaxName string Required Primary passenger name (e.g. "JITHIN"). Required for all sources.
strCountry string Optional Destination/service country name (e.g. "Germany"). Optional — if provided, must match an existing country in NutraACS. No auto-creation. Must be consistent with strRegionCode.
strCity string Optional City name (e.g. "Berlin"). Optional — if provided, must exist in NutraACS and must belong to strCountry (when both are given).
strPnrNumber string Optional GDS PNR or booking reference number. Max 15 characters.
strGdsCompany string Optional GDS platform used for booking. Max 20 characters. Accepted values (case-insensitive): AMADEUS, GALILEO, SABRE, WORLDSPAN, FARELOGIX, ONLINE, LCC, AIRLINE, OTHER. Defaults to "Other" if not provided.
strSupplierConfirmNo string Optional Supplier's confirmation number. Max 50 characters. Must be unique across the system — duplicates are rejected per-voucher.
strReferenceNumber string Optional Internal or external reference number (e.g. "46121"). Max 50 characters.
strCustomerEmpNo string Optional Customer employee number for corporate identification (e.g. "RAJEEV@21423"). Stored against the voucher master.
datBooking string (date) Optional Date the service was booked. Format: "DD/MM/YYYY". If provided, must be a valid date.
strBookingStaffId string Optional Staff code of the booking agent. Resolved to a staff account. Falls back to strBookingStaffEmail if absent (THIRD_PARTY source only).
strBookingStaffEmail string Optional Email of the booking agent. Used as fallback lookup when strBookingStaffId is absent (THIRD_PARTY source only).
strIssuingStaffId string Optional Staff code of the issuing agent. Optional for THIRD_PARTY source. Falls back to strIssuingStaffEmail if absent.
strIssuingStaffEmail string Optional Email of the issuing agent. Fallback lookup when strIssuingStaffId is absent (THIRD_PARTY source only).
strBranch string Optional Branch code where the voucher is posted (e.g. "MAIN"). Must match an active branch in NutraACS. Optional for THIRD_PARTY source.
strBookingDetails string Optional Free-text booking details or notes stored against the voucher.
strAdditionalReference string Optional Additional reference or internal approval code. Max 200 characters.
strTermsAndConditions string Optional Terms and conditions text stored against the voucher.
objDimensions object Optional Key-value object for custom dimension metadata at voucher level.
Unused fields in example payload:
  • intServiceType — this field is not read by the API. Use strServiceType (the service name) instead.

Financial, supplier and customer data — all inside objDetails

All amount fields should be numbers or numeric strings. Empty string "" is treated as zero/not provided. Card and GR fields (strMaskedCardNo, etc.) must be inside objDetails.

strCustomerCurrency string Optional ISO 4217 currency code for customer-facing amounts. If not provided for THIRD_PARTY, defaults to the system base currency automatically.

dblServiceFee1 number Optional Primary service fee charged to the customer (maps to dblServiceFeeCustomer).
dblServiceFee2 number Optional Extra earning / secondary fee (maps to dblExtraEarningCustomer).
dblServiceFee3 number Optional Additional income / tertiary fee (maps to dblAdditionalIncomeCustomer).

Fields NOT supported (present in some example payloads but ignored by the API):
  • dblInputVatOnSupplierFee — not mapped in the usecase, will be ignored.
  • dblTcsInput — pending implementation (TODO), currently ignored.
  • dblSupChargeRfd — not mapped, will be ignored.
  • dblCustomerExpenseOutputTax — incorrect alias; use dblExpenseOutputTax.

4c. GDS Remarks & Transactional Remarks

Both objects are placed at the voucher root level (not inside objDetails).

GDS Remarks — objGdsRemarks

An optional key-value object for structured GDS remark capture. Each key is a GDS remark integration code configured in NutraACS for the OTHER service type and the value is the remark text. The system validates input type (ALPHABET, NUMERIC, ALPHANUMERIC, FREE_TEXT) and max length per remark configuration.

"objGdsRemarks": {
  "STRCCNO":         "89598",
  "AAASSASASASASAS": "asdsadsad"
}
Input TypeAllowed Characters
ALPHABETLetters (A–Z, a–z) and spaces only
NUMERICDigits (0–9) only
ALPHANUMERICLetters, digits, and spaces only
FREE_TEXTNo restriction
ValidationError Code
Mandatory GDS remark not provided IMPORTING_ERRORS.KEY_MISSING_GDS_REMARK_{systemAttributeName}
Remark value does not match the configured input type IMPORTING_ERRORS.INVALID_INPUT_TYPE_GDS_REMARK_{systemAttributeName}
Remark value exceeds the configured max length IMPORTING_ERRORS.MAX_LENGTH_EXCEEDED_GDS_REMARK_{systemAttributeName}

Transactional Remarks — objTransactionalRemarks

An optional key-value object for configurable transactional remarks. The key is the remark master name as configured in NutraACS and the value is the detail option's code or name.

Do NOT send primary key IDs. Send the human-readable master name as the key and the detail's vchr_code or vchr_name as the value. The system tries vchr_code first (case-insensitive), then falls back to vchr_name. 
"objTransactionalRemarks": {
  "TESTTI": "abc",
  "TEST":   "1234"
}
ValidationError Code
Mandatory remark master has no value supplied IMPORTING_ERRORS.TRANSACTIONAL_REMARKS_MANDATORY_MISSING_{masterPk}
Master name does not exist in NutraACS IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_MASTER_{masterName}
Detail code/name is not a valid option for the given master IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_DETAIL_{masterName}

4d. Dimension Codes

Up to three optional dimension codes at the voucher root level. Dimensions are only enforced when both the dimension feature flag and the coupon-level dimension setting are active in tenant configuration.

FieldTypeRequiredDescription
strDimensionCodeOne string Cond. Code for Dimension 1 (e.g. "d1", "CC-001"). Required if Dimension 1 is active at coupon level. Values are uppercased before lookup.
strDimensionCodeTwo string Cond. Code for Dimension 2. Required if Dimension 2 is active at coupon level.
strDimensionCodeThree string Cond. Code for Dimension 3. Required if Dimension 3 is active at coupon level.

5. Payment Info (objPaymentInfo)

When objPaymentInfo is provided, all successfully saved vouchers are invoiced as a batch. Payment validation runs before saving — if it fails, no vouchers are saved.

FieldTypeRequiredDescription
strDocumentDate string (date) Required Invoice document date. Format: "DD/MM/YYYY".
strModeOfSale string Required Mode of sale (e.g. "Credit", "Cash").
arrPaymentOptions array Required One or more payment option objects. See below.

Payment Option Object (arrPaymentOptions[*])

FieldTypeRequiredDescription
intMasterCustomer string / number Required Primary key of the master customer account in NutraACS.
strPaymentType string Required Payment type (e.g. "Credit", "Cash", "Cheque", "CreditCard").
strCustomerCode string Required Customer account code being invoiced.
strCurrency string Required ISO 4217 currency code for the payment amount.
dblTranCurDebitedAmount number Required Total amount collected in the transaction currency.
dblTranCurrRoe number Optional Rate of exchange from transaction currency to base currency. Defaults to 1.
intPaymentOptionPk string / number Optional Payment option configuration PK. Leave empty for standard flows.
strCreditCardNo string Optional Credit card number (for card payment type).
strChequeNo string Optional Cheque number (for cheque payment type).
datPayment string (date) Optional Payment date. Format: "DD/MM/YYYY".
datCheque string (date) Optional Cheque date. Format: "DD/MM/YYYY".
dblTranCurBankCharge number Optional Bank charge in transaction currency.
dblBaseCurBankCharge number Optional Bank charge in base currency.
objPOSDetails object Optional Point-of-sale terminal details (for card transactions).
strBatchNo string Optional Card payment batch number for reconciliation.
strCCRecoNo string Optional Corporate card reconciliation number.

6. Validations

6a. Batch-Level Validations (rejects entire request)

RuleError Response
One or more vouchers missing strVoucherNo { "strMessage": "KEY_MISSING_TICKET_NUMBER" }
One or more vouchers missing strTransactionType { "strMessage": "KEY_MISSING_TRANSACTION_TYPE" }
strTransactionType is not "SALE" or "REFUND" { "strMessage": "INVALID_TRANSACTION_TYPE" }
objPaymentInfo is provided but payment validation fails { "strMessage": "ERROR_IN_PAYMENT_INFORMATION", "objDetails": { "arrMessages": [...] } }
No vouchers remain after validation { "strMessage": "NO_RECORD_TO_SYNC" }
