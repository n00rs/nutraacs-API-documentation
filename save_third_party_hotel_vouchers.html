NutraACS Integration Guide – Save Third Party Hotel Vouchers API

This API allows third-party providers (e.g. hotel booking platforms, GDS systems, OTA portals) to push hotel voucher transactions directly into NutraACS for both Sale and Refund processing.

1. API Endpoint

POST /api/voucher/hotel/save_third_party_hotel_vouchers

Content-Type: application/json

2. Request Structure

{
  "strProviderCode": "PROVIDERCODE",
  "objPaymentInfo": { ... },
  "arrVouchers": [
    {
      "strVoucherNo": "HV25001",
      "strTransactionType": "SALE",
      "strVoucherStatus": "I",
      "strHotelName": "101 Hotel",
      "strHotelCode": "9IO",
      ...
      "objDetails": { ... },
      "objGdsRemarks": { ... },
      "objTransactionalRemarks": { ... }
    }
  ]
}

3. Top-Level Fields

FieldTypeRequiredDescription
strProviderCode string Required The unique vendor/provider code identifying the third-party system (e.g. "NUFLIGHTS"). Must be pre-registered in NutraACS.
arrVouchers array Required Array of one or more hotel voucher objects to import. Each object represents a single hotel transaction (Sale or Refund). See Section 4.
objPaymentInfo object Optional Payment details for batch invoicing. When provided, all successfully saved vouchers are automatically invoiced. See Section 5.

4. Voucher Object Fields

Each element in arrVouchers represents one hotel transaction. Fields are split into root-level identifiers, Master (hotel & booking info) and Details (financial & occupancy data inside objDetails).

4a. Master Fields (voucher root level)

Hotel, location, guest and booking information

FieldTypeRequiredDescription & Validation
strVoucherNo string Required Unique hotel voucher number (e.g. "HV25001"). Max 30 characters. Only letters, digits, / and - are allowed — special characters cause a validation error. Cannot be null or empty; the entire batch is rejected if missing.
strTransactionType string Required Type of transaction. Accepted values: "SALE" or "REFUND" (case-sensitive). Any other value rejects the entire batch.
strVoucherStatus string Required Current status of the voucher. Accepted values:
  • "I" or "ISSUE" — Issued (use with SALE)
  • "R" or "REFUND" — Refunded (use with REFUND)
  • "V" or "VOID" — Voided (use with REFUND)
Status must be consistent with strTransactionType. Issue status only for SALE; Refund/Void only for REFUND. Mismatch causes a per-voucher error.
strHotelName string Cond. Hotel name as registered in NutraACS (e.g. "101 Hotel"). Max 200 characters. For third-party requests, either strHotelName or strHotelCode must be provided. When both are provided, they must match the same hotel record. If the hotel doesn't exist it is auto-created (subject to length constraints).
strHotelCode string Cond. Hotel code as registered in NutraACS (e.g. "9IO"). Max 50 characters. Used as an alternative or supplement to strHotelName for hotel resolution. One of the two must be present for THIRD_PARTY source.
strHotelCountry string Required Country name where the hotel is located (e.g. "Iceland"). Max 100 characters. Used to determine domestic vs. international status. If the country doesn't exist, it is auto-created. Must pair consistently with strCountryCode.
strCountryCode string Optional ISO country code for the hotel's country (e.g. "IS"). Max 5 characters. Used alongside strHotelCountry for country resolution and auto-creation. Must be consistent — a code already linked to a different country name causes an error.
strCity string Required City name where the hotel is located (e.g. "Reykjavik"). Max 100 characters. Must belong to the provided strHotelCountry. Also validated against the hotel's configured city if the hotel is found — mismatch causes an error.
strOriginCountry string Optional Country of origin of the passenger/guest (e.g. "India"). Optional for THIRD_PARTY source. If provided, it must be a valid country in NutraACS.
strHotelRoomType string Optional Room type name as configured in NutraACS (e.g. "1 Pax Shared Dorm with Bkfst"). Max 200 characters. Optional for THIRD_PARTY source — if provided, it must be a valid room type. Auto-created if not found (subject to max length).
strMealsPlan string Optional Meal plan name as configured in NutraACS (e.g. "All Inclusive", "Bed and Breakfast"). Max 200 characters. Optional for THIRD_PARTY source — auto-created if not found.
strGuestName string Optional Primary guest name as it appears on the booking (e.g. "JITHIN K").
strAdditionalGuests string Optional Names of additional guests, comma-separated or as a free-text string.
strPaxType string Optional Passenger type (e.g. "ADT", "CHD"). Stored against the master record.
strPnrNumber string Optional GDS PNR / booking reference number. Max 15 characters.
strGdsCompany string Optional GDS platform used to book the hotel. Max 20 characters. Accepted values (case-insensitive): AMADEUS, GALILEO, SABRE, WORLDSPAN, FARELOGIX, ONLINE, LCC, AIRLINE, OTHER. Defaults to "Other" if not provided. Title-cased before saving.
strHotelConfirmNo string Optional Hotel's booking confirmation number. Max 50 characters. Must be unique — duplicate confirmation numbers are rejected.
strSupplierConfirmNo string Optional Supplier's confirmation number. Max 50 characters. Must be unique across the system.
datCheckIn string (date) Optional Check-in date. Format: "DD/MM/YYYY" or "YYYY-MM-DD".
datCheckOut string (date) Optional Check-out date. Same format as datCheckIn.
strBookingStaffId string Optional Staff code of the booking agent. Resolved to the staff account in NutraACS. Falls back to strBookingStaffEmail if absent (THIRD_PARTY source only).
strBookingStaffEmail string Optional Email of the booking agent. Used as fallback lookup when strBookingStaffId is absent (THIRD_PARTY source only).
strBookingDetails string Optional Free-text booking details or notes. Stored against the voucher.
strAdditionalReference string Optional Additional reference text (e.g. internal approval code or project reference). Max 200 characters.
strTermsAndConditions string Optional Terms and conditions text stored against the voucher.
strRegionCode string Optional Tax region code (e.g. "INT" for international, "DOM" for domestic). If not provided, region is auto-determined by comparing the hotel country with the company's country.
strPickUpTerminal string Optional Airport terminal or identifier for hotel transfer pick-up.
strPickUpFlightNo string Optional Incoming flight number for hotel pick-up coordination.
strPickUpAirline string Optional Airline code/name for the incoming pick-up flight.
datPickUp string (date) Optional Scheduled pick-up date. Format: "DD/MM/YYYY".
strDropOffTerminal string Optional Terminal identifier for hotel transfer drop-off.
strDropOffFlightNo string Optional Outbound flight number for hotel drop-off coordination.
strDropOffAirline string Optional Airline code/name for the outbound drop-off flight.
datDropOff string (date) Optional Scheduled drop-off date. Format: "DD/MM/YYYY".
objDimensions object Optional Key-value object for custom dimension metadata at voucher level. Structure defined by tenant configuration.

4b. Details Fields (objDetails)

Occupancy, financial, supplier and customer data

All amount fields should be numbers or numeric strings. Empty string "" is treated as zero/not provided. Card and GR fields (strMaskedCardNo, etc.) belong inside objDetails, where the system reads them.
FieldTypeRequiredDescription & Validation
datIssue string (date) Cond. Required when strTransactionType is "SALE". The voucher issue date. Format: "DD/MM/YYYY". For REFUND, datRefund is used instead.
datRefund string (date) Cond. Required when strTransactionType is "REFUND". The refund processing date. Must be on or after the original sale issue date. Same format as datIssue.
strRatePerNight string / number Optional Room rate per night in supplier currency. Must be a valid numeric value if provided. Optional for THIRD_PARTY source.
strNoOfAdults string / number Optional Number of adult guests (e.g. "1"). Must be a positive integer if provided. Optional for THIRD_PARTY source.
strNoOfChildren string / number Optional Number of child guests. Must be a non-negative integer if provided.
strNoOfInfant string / number Optional Number of infant guests. Must be a non-negative integer if provided.
strNoOfRooms string / number Optional Number of rooms booked (e.g. "1"). Must be a positive integer if provided.
strNoOfNights string / number Optional Number of nights of the stay. Must be a positive integer if provided.
strBranch string Optional Branch code where the voucher is posted (e.g. "MAIN"). Must match an active branch in NutraACS. Optional for THIRD_PARTY source.
strIssuingStaffId string Optional Staff code of the issuing agent. Resolved to the staff account. Falls back to strIssuingStaffEmail if absent (THIRD_PARTY only).
strIssuingStaffEmail string Optional Email of the issuing agent. Fallback lookup when strIssuingStaffId is absent (THIRD_PARTY source only).
strSupplier string Optional Supplier account code in NutraACS (the hotel supplier billed by the agency).
strSupplierCurrency string Optional ISO 4217 currency code for supplier-side amounts (e.g. "INR", "USD"). Must be configured in NutraACS.
dblSupplierROE number Optional Supplier Rate of Exchange — conversion from supplier currency to base currency.
dblPublishedFare number Optional Published (rack) hotel rate in supplier currency before taxes. Also accepted via dblSupplierDebit / dblSupplierCredit if this field is zero.
dblMarketFare number Optional Negotiated/market hotel rate in supplier currency.
dblMarketFareCardAmount number Optional Portion of market fare charged to a corporate card.
dblTax number Optional Total tax on the hotel booking in supplier currency.
dblTaxCardAmount number Optional Portion of tax charged to a card.
strTaxDetails string Optional Raw tax breakdown string. Max 250 characters.
dblSupplierCommission1 number Optional Standard commission from the supplier (maps to dblStdCommission).
dblSupplierCommission2 number Optional PLB commission from the supplier (maps to dblPlbCommission).
dblInputTax number Optional Input (recoverable) tax on supplier charges (e.g. GST input credit).
dblOtherInputTax number Optional Any additional input tax not covered by dblInputTax.
dblSupplierFee number Optional Processing/service fee charged by the supplier to the agency.
dblSupplierFeeTax number Optional Tax applied on the supplier fee.
dblSupplierCharge number Optional Additional supplier-imposed charge (e.g. early departure penalty).
dblSupplierChargeCard number Optional Portion of the supplier charge paid by card.
dblIgst number Optional Integrated GST (India — inter-state transactions).
dblSgst number Optional State GST (India — intra-state transactions).
dblCgst number Optional Central GST (India — intra-state transactions).
dblOutputIgst number Optional Output IGST charged to the customer.
dblOutputSgst number Optional Output SGST charged to the customer.
dblOutputCgst number Optional Output CGST charged to the customer.
strCorpCardCode string Optional Corporate credit card account code in NutraACS. Used when the hotel was paid via a configured corporate card.
strMaskedCardNo string Optional Masked credit/debit card number used for payment (e.g. "1347****7854"). Do NOT send full unmasked card numbers.
strCardAuthorizationCode string Optional Payment gateway authorization code for the card transaction (e.g. "7CA79854").
strGrAccountCode string Optional Government Receivable (GR) account code for government-billed transactions.
strGrLpoNumber string Optional LPO number associated with the GR account.
strFopElement string Optional Form of Payment element from the GDS PNR (e.g. "CC VI4111111111111111/0125").
strCustomer string Optional Customer account code in NutraACS. The voucher is billed to this customer. Must be a valid, active customer account.
strCustomerCurrency string Optional ISO 4217 currency code for customer-facing amounts. Defaults to the customer's configured currency if not provided.
dblCustomerROE number Optional Rate of Exchange from customer currency to the system base currency.
strCustomerLpoOrToNumber string Optional Customer's LPO or Tour Order reference number for corporate billing. Max 50 characters.
datCustomerLpoDate string (date) Optional Date of the customer LPO. Format: "DD/MM/YYYY".
dblServiceFee1 number Optional Primary service fee charged to the customer (maps to dblServiceFeeCustomer).
dblServiceFee2 number Optional Extra earning / secondary fee charged to the customer (maps to dblExtraEarningCustomer).
dblServiceFee3 number Optional Additional income / tertiary fee (maps to dblAdditionalIncomeCustomer).
dblDiscount1 number Optional Primary discount given to the customer (maps to dblDiscountCustomer).
dblDiscount2 number Optional PLB discount passed to the customer (maps to dblPlbDiscountCustomer).
dblDiscount3 number Optional Additional / tertiary discount (maps to dblAdditionalDiscountCustomer).
dblAgencyChargeRfd number Optional Agency-retained cancellation/refund fee charged to the customer on refund transactions.
dblCustomerOutputTax number Optional Output VAT / billing tax charged to the customer on top of the selling price.
dblExpenseOutputTax number Optional Output tax on the agency expense side (maps to dblCustomerExpenseOutputTax).
dblSellingPriceCustomer number Optional Total selling price billed to the customer in customer currency (fare + taxes + fees − discounts).
strPaybackAccount1 string Optional GL account code for the first payback/rebate distribution.
dblPaybackAmount1 number Optional Amount to distribute to strPaybackAccount1.
strPaybackAccount2 string Optional GL account code for the second payback account.
dblPaybackAmount2 number Optional Amount to distribute to strPaybackAccount2.
strPaybackAccount3 string Optional GL account code for the third payback account.
dblPaybackAmount3 number Optional Amount to distribute to strPaybackAccount3.
Note on card/GR fields: strMaskedCardNo, strCardAuthorizationCode, strGrAccountCode, strGrLpoNumber, and strFopElement must be placed inside objDetails. The system reads these from objDetails and stores them in the voucher extras. Placing them at the voucher root level will result in them being ignored.

4c. GDS Remarks & Transactional Remarks

GDS Remarks — objGdsRemarks

An optional key-value object for structured GDS remark capture. Each key is a GDS remark integration code configured in NutraACS for the HOTEL service type and the value is the remark text. The system validates input type (ALPHABET, NUMERIC, ALPHANUMERIC, FREE_TEXT) and max length per remark configuration.

"objGdsRemarks": {
  "STRCCNO":   "89598",
  "TOUR_CODE": "CORP2024"
}
ValidationError Code
Mandatory GDS remark not provided IMPORTING_ERRORS.KEY_MISSING_GDS_REMARK_{systemAttributeName}
Remark value does not match the configured input type IMPORTING_ERRORS.INVALID_INPUT_TYPE_GDS_REMARK_{systemAttributeName}
Remark value exceeds the configured max length IMPORTING_ERRORS.MAX_LENGTH_EXCEEDED_GDS_REMARK_{systemAttributeName}

Transactional Remarks — objTransactionalRemarks

An optional key-value object for configurable transactional remark capture. The key is the remark master name as configured in NutraACS and the value is the detail option's code or name.

Do NOT send primary key IDs. Send the human-readable master name as the key and the detail's vchr_code or vchr_name as the value. The system tries vchr_code first (case-insensitive), then falls back to vchr_name. 
"objTransactionalRemarks": {
  "TESTTI": "abc",
  "TEST":   "1234"
}
ValidationError Code
Mandatory remark master has no value supplied IMPORTING_ERRORS.TRANSACTIONAL_REMARKS_MANDATORY_MISSING_{masterPk}
Master name does not exist in NutraACS IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_MASTER_{masterName}
Detail code/name is not a valid option for the given master IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_DETAIL_{masterName}

4d. Dimension Codes

Up to three optional dimension codes can be attached per voucher. Dimensions are only enforced when both the dimension feature flag and the coupon-level dimension setting are active in tenant configuration. If active and a mandatory dimension code is missing, the voucher fails validation.

FieldTypeRequiredDescription
strDimensionCodeOne string Cond. Code for Dimension 1 (e.g. cost centre "CC-001"). Required if Dimension 1 is active at coupon level. Values are uppercased before lookup.
strDimensionCodeTwo string Cond. Code for Dimension 2 (e.g. project code "PROJ-ALPHA"). Required if Dimension 2 is active at coupon level.
strDimensionCodeThree string Cond. Code for Dimension 3. Required if Dimension 3 is active at coupon level.

5. Payment Info (objPaymentInfo)

When objPaymentInfo is provided, all successfully saved vouchers are invoiced as a batch. Payment validation runs before saving — if it fails, no vouchers are saved.

FieldTypeRequiredDescription
strDocumentDate string (date) Required Invoice document date. Format: "DD/MM/YYYY".
strModeOfSale string Required Mode of sale (e.g. "Credit", "Cash").
arrPaymentOptions array Required One or more payment option objects. See payment option fields below.

Payment Option Object (arrPaymentOptions[*])

FieldTypeRequiredDescription
intMasterCustomer string / number Required Primary key of the master customer account in NutraACS.
strPaymentType string Required Payment type (e.g. "Credit", "Cash", "Cheque", "CreditCard").
strCustomerCode string Required Customer account code being invoiced. Must match the voucher's customer.
strCurrency string Required ISO 4217 currency code for the payment amount.
dblTranCurDebitedAmount number Required Total amount collected in the transaction currency.
dblTranCurrRoe number Optional Rate of exchange from transaction currency to base currency. Defaults to 1.
intPaymentOptionPk string / number Optional Payment option configuration PK. Leave empty for standard flows.
strCreditCardNo string Optional Credit card number (for card payment type).
strChequeNo string Optional Cheque number (for cheque payment type).
datPayment string (date) Optional Payment date. Format: "DD/MM/YYYY".
datCheque string (date) Optional Cheque date. Format: "DD/MM/YYYY".
dblTranCurBankCharge number Optional Bank charge in transaction currency.
dblBaseCurBankCharge number Optional Bank charge in base currency.
objPOSDetails object Optional Point-of-sale terminal details (for card transactions).
strBatchNo string Optional Card payment batch number for reconciliation.
strCCRecoNo string Optional Corporate card reconciliation number.

6. Validations

6a. Batch-Level Validations (rejects entire request)

RuleError Response
One or more vouchers missing strVoucherNo { "strMessage": "KEY_MISSING_TICKET_NUMBER" }
One or more vouchers missing strTransactionType { "strMessage": "KEY_MISSING_TRANSACTION_TYPE" }
strTransactionType is not "SALE" or "REFUND" { "strMessage": "INVALID_TRANSACTION_TYPE" }
objPaymentInfo is provided but payment validation fails { "strMessage": "ERROR_IN_PAYMENT_INFORMATION", "objDetails": { "arrMessages": [...] } }
No vouchers remain after validation { "strMessage": "NO_RECORD_TO_SYNC" }

6b. Per-Voucher Validations (reported in import result)

Failed vouchers are moved to the error list while remaining valid vouchers are still processed.

8. Responses

Success

{
  "Success": {
    "Sale":   ["HV25001"],
    "Refund": []
  },
  "Error": {
    "Sale":   {},
    "Refund": {}
  },
  "arrPaymentErrors": [],
  "strDocumentNumber": "INV-2026-00124"
}
strDocumentNumber is only returned when objPaymentInfo was provided and the invoice was successfully generated.
