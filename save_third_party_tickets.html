NutraACS Integration Guide – Save Third Party Tickets API

This API allows third-party providers (e.g. airlines, GDS systems, OTA platforms) to push flight ticket transactions directly into NutraACS for both Sale and Refund processing.

1. API Endpoint

POST /api/voucher/ticket/save_third_party_tickets

Content-Type: application/json

2. Request Structure

{
  "strProviderCode": "NUFLIGHTS",
  "objPaymentInfo": { ... },
  "arrTickets": [
    {
      "strTicketNumber": "176-2345678901",
      "strTransactionType": "SALE",
      ...
      "objDetails": { ... },
      "arrSegmentDetails": [ ... ],
      "objGdsRemarks": { ... },
      "objTransactionalRemarks": { ... }
    }
  ]
}

3. Top-Level Fields

FieldTypeRequiredDescription
strProviderCode string Required The unique vendor/provider code that identifies the third-party system sending the data (e.g. "NUFLIGHTS", "ETHIOPIAN"). Must be registered in NutraACS beforehand.
arrTickets array Required Array of one or more ticket objects to import. Each object represents a single ticket transaction (Sale or Refund). See Section 4 for field details.
objPaymentInfo object Optional Payment details applicable across the batch (e.g. bank account, payment mode). Structure defined by the tenant's payment configuration.

4. Ticket Object Fields

Each element in arrTickets represents one ticket. Fields are organized into: top-level identifiers, Master data (passenger & ticket info), Details data (financial & operational), and Extras (supplementary references).

4a. Master Fields (ticket-level)

Passenger, airline, and booking information

FieldTypeRequiredDescription & Validation
strTicketNumber string Required The airline ticket number (e.g. "176-2345678901"). Must be unique per transaction. The system converts this to uppercase. Cannot be null or empty — the entire batch is rejected if any ticket is missing this field.
strTransactionType string Required Indicates whether this is a new sale or a refund. Accepted values: "SALE" or "REFUND" (case-sensitive). Any other value causes the entire batch to be rejected.
strPaxName string Optional Passenger full name as it appears on the ticket (e.g. "SMITH/JOHN MR"). Multiple names can be comma-separated. Duplicates (case-insensitive) are automatically removed before saving.
strPaxType string Optional Passenger type code. Standard IATA codes accepted: ADT (Adult), CHD (Child), INF (Infant without seat), INS (Infant with seat), UNN (Unaccompanied minor).
strTicketType string Optional Type of document. Use "EMD" for Electronic Miscellaneous Documents. For EMD tickets, the strRemarks or strSector value is also stored as an EMD remark and auto-created if not present in the system.
strTicketStatus string Optional Current ticket status as reported by the GDS or airline (e.g. "OPEN", "USED", "VOID").
strGdsCompany string Optional GDS platform used to issue the ticket. Accepted values: AMADEUS, GALILEO, SABRE, WORLDSPAN, FARELOGIX. Used to resolve the correct agent codes (numeric / character) for booking and ticketing agents.
strPnrNumber string Optional GDS Passenger Name Record (booking reference) number (e.g. "ABC123").
datBooking string (date) Optional Date the booking was made. Accepts ISO 8601 format ("YYYY-MM-DD") or "DD/MM/YYYY".
datTravelDate string (date) Optional Outbound travel date. Format: "YYYY-MM-DD" or "DD/MM/YYYY".
datReturnDate string (date) Optional Return travel date for round-trip tickets. Same date format as datTravelDate.
strClass string Optional Cabin class for the outbound journey (e.g. "Economy", "Business", "First").
strClassRBD string Optional Reservation Booking Designator (fare class code) for the outbound journey (e.g. "Y", "K", "B").
strReturnClass string Optional Cabin class for the return journey.
strReturnRBD string Optional RBD (fare class code) for the return journey.
intAirlineNumericCode string / number Optional The 3-digit IATA numeric airline code (e.g. "176" for Ethiopian Airlines). Used to resolve the airline account in NutraACS.
strRegionCode string Cond. Tax region code. Mandatory for EMD ticket types. Used to apply the correct tax region rules to the transaction.
strOriginalIssue string Optional Original issue information for reissued tickets (e.g. the original ticket number or original issuance details).
strRemarks string Optional Free-text remarks to be attached to the ticket. For EMD tickets, this value (or strSector if empty) is also stored as an EMD remark.
strBookingAgentId string Optional Identifier of the booking agent/staff. Resolved to the agent's NutraACS account. The exact code type used (Amadeus, Galileo, etc.) is determined by strGdsCompany.
strBookingAgentEmail string Optional Email address of the booking agent. Used as an alternative lookup key if strBookingAgentId is not resolvable.
strConnectionTicketNumber string Optional Ticket number of the connecting flight ticket that this ticket is linked to.
objDimensions object Optional Key-value object for custom dimension metadata attached at the ticket level (e.g. cost centres, project codes). Structure defined by tenant configuration.

4b. Details Fields (objDetails)

Financial, supplier, and customer fare data

All financial amounts should be provided as numbers (or numeric strings). Currency amounts must be consistent with the corresponding currency fields (strSupplierCurrency / strCustomerCurrency).
FieldTypeRequiredDescription & Validation
datIssue string (date) Cond. Required when strTransactionType is "SALE". The ticket issue date. Format: "YYYY-MM-DD" or "DD/MM/YYYY". Not used for refunds.
datRefund string (date) Cond. Required when strTransactionType is "REFUND". The date the refund was processed. Same date format as datIssue.
strBranch string Optional Branch code where the ticket is being posted. Resolved to the branch account in NutraACS. If omitted, the default branch of the customer is used.
strTicketingAgentId string Optional Identifier of the ticketing agent/staff who issued the ticket. Resolved using strGdsCompany to pick the correct code type.
strTicketingAgentEmail string Optional Email of the ticketing agent. Used as fallback lookup if strTicketingAgentId is not resolvable.
strSector string Optional Flight routing in sector format (e.g. "DXB/BOM/DXB"). For EMD tickets, used as the EMD remark text if strRemarks is empty. The system counts / separators to derive number of segments.
strLastConjunctionTicketNumber string Optional The last ticket number in a conjunction ticket sequence. Presence of this field marks the ticket as a conjunction ticket.
strSupplier string Optional Supplier code (airline or consolidator account code in NutraACS) who issued the ticket.
strSupplierCurrency string Optional ISO 4217 currency code for supplier-side amounts (e.g. "USD", "AED", "INR"). Must be configured in NutraACS.
dblSupplierROE number Optional Supplier Rate of Exchange — conversion rate from supplier currency to the base (system) currency. Used in fare calculations.
dblPublishedFare number Optional The published (base) fare amount in supplier currency before taxes and charges.
dblMarketFare number Optional Market/selling fare in supplier currency (may differ from published fare due to negotiated deals).
dblMarketFareCardAmount number Optional Portion of the market fare paid by corporate/credit card.
dblTax number Optional Total airline taxes amount in supplier currency.
dblTaxCardAmount number Optional Portion of the tax paid by card.
strTaxDetails string Optional Raw tax breakdown string as provided by the GDS (e.g. "YQ15.00USGB12.50").
dblSupplierCommission1 number Optional Standard commission amount from the airline/supplier (maps to dblStdCommission in NutraACS).
dblSupplierCommission2 number Optional PLB (Performance Linked Bonus) commission amount from the airline/supplier (maps to dblPlbCommission).
dblAirlineCharge number Optional Additional airline-imposed charge (e.g. rebooking fee, change fee) in supplier currency.
dblAirlineChargeCard number Optional Portion of the airline charge paid by card.
dblInputTax number Optional Input (recoverable) tax applicable on supplier-side charges (e.g. GST input credit).
dblOtherInputTax number Optional Any additional input tax amounts not covered by dblInputTax.
dblSupplierFee number Optional Service/processing fee charged by the supplier to the agency.
dblSupplierFeeTax number Optional Tax applied on the supplier fee.
dblSupplierCharge number Optional Additional supplier-imposed charge on the transaction (e.g. credit card surcharge billed to agency).
dblSupplierChargeCard number Optional Portion of supplier charge paid by card.
dblIgst number Optional Integrated GST amount (applicable for Indian tax regime — inter-state transactions).
dblSgst number Optional State GST amount (applicable for Indian tax regime — intra-state transactions).
dblCgst number Optional Central GST amount (applicable for Indian tax regime — intra-state transactions).
dblOutputIgst number Optional Output IGST charged to the customer (Indian GST — inter-state).
dblOutputSgst number Optional Output SGST charged to the customer (Indian GST — intra-state).
dblOutputCgst number Optional Output CGST charged to the customer (Indian GST — intra-state).
strCorpCardCode string Optional Corporate credit card account code in NutraACS. Used when the ticket was paid partially or fully via a corporate card.
strCustomer string Optional Customer account code in NutraACS (e.g. "IN0098"). The ticket will be billed to this customer. Must be a valid, active customer code.
strCustomerCurrency string Optional ISO 4217 currency code for customer-facing amounts. Defaults to the customer's configured currency if not provided.
dblCustomerROE number Optional Rate of Exchange from customer currency to the system base currency. Used to calculate customer-side amounts in base currency.
dblServiceFee1 number Optional Primary service fee charged to the customer (maps to dblServiceFeeCustomer).
dblServiceFee2 number Optional Extra earning / secondary service fee charged to the customer (maps to dblExtraEarningCustomer).
dblServiceFee3 number Optional Additional income / tertiary fee charged to the customer (maps to dblAdditionalIncomeCustomer).
dblDiscount1 number Optional Primary discount given to the customer (maps to dblDiscountCustomer).
dblDiscount2 number Optional PLB discount passed to the customer (maps to dblPlbDiscountCustomer).
dblDiscount3 number Optional Additional / tertiary discount for the customer (maps to dblAdditionalDiscountCustomer).
dblSellingPriceCustomer number Optional Total selling price (fare + taxes + fees − discounts) billed to the customer in customer currency.
dblCustomerOutputTax number Optional Output VAT / billing tax amount charged to the customer on top of the selling price.
dblExpenseOutputTax number Optional Output tax on agency expense side (maps to dblCustomerExpenseOutputTax).
dblAgencyChargeRfd number Optional Agency-retained charge on a refund transaction (e.g. cancellation fee retained by the agency).
strCustomerLpoOrToNumber string Optional Customer's Local Purchase Order (LPO) or Tour Order reference number. Used for corporate billing reconciliation.
datCustomerLpoDate string (date) Optional Date of the customer LPO. Format: "YYYY-MM-DD".
strPaybackAccount1 string Optional Account code for the first payback/rebate account (used to distribute part of the income to another GL account).
dblPaybackAmount1 number Optional Amount to pay back to strPaybackAccount1.
strPaybackAccount2 string Optional Account code for the second payback account.
dblPaybackAmount2 number Optional Amount to pay back to strPaybackAccount2.
strPaybackAccount3 string Optional Account code for the third payback account.
dblPaybackAmount3 number Optional Amount to pay back to strPaybackAccount3.
strMaskedCardNo string Optional Masked credit/debit card number used for payment (e.g. "4111XXXXXXXX1111"). Do NOT send the full unmasked card number.
strCardAuthorizationCode string Optional Payment gateway authorization code for the card transaction.
strGrAccountCode string Optional Government Receivable (GR) account code for government-billed transactions.
strGrLpoNumber string Optional LPO number associated with the GR account.
strFopElement string Optional Form of Payment (FOP) element as stored in the GDS PNR (e.g. "CC VI4111111111111111/0125").

4c. Extras Fields (ticket root level)

Supplementary references stored in ticket extras

FieldTypeRequiredDescription
strIataNo string Optional IATA agency number associated with the issuing agency for this ticket.
strGdsPnrNumber string Optional GDS-specific PNR number (may differ from strPnrNumber when the GDS booking reference uses a different format).
strRelatedTicketNo string Optional Ticket number of a related/original ticket (e.g. the original ticket being exchanged or refunded).
strPaxEmail string Optional Passenger email address. Used for communication and stored in ticket extras.
strPaxContactNo string Optional Passenger phone / contact number.

4d. Segment Details (arrSegmentDetails)

Individual flight segments for the ticket (applicable to SALE transactions)

arrSegmentDetails is an optional array of flight segments. Each segment represents one flight leg. Segment data is only saved for SALE transactions; it is ignored for REFUND.
FieldTypeRequiredDescription
strFlightNumber string Optional Flight number (e.g. "ET 303").
strOpsAirlineNumCode string Optional Operating airline's 3-digit IATA numeric code.
strOpsAirlineChrCode string Optional Operating airline's 2-letter IATA character code (e.g. "ET").
strDepartureDate string (datetime) Optional Departure date and time. Accepts ISO 8601 or standard date strings. The date portion is stored as dat_departure_date; the full string as departure time.
strArrivalDate string (datetime) Optional Arrival date and time. Same format as strDepartureDate.
strOriginType string Optional IATA 3-letter origin airport code (e.g. "DXB").
strArrivalAirport string Optional IATA 3-letter destination/arrival airport code (e.g. "BOM"). Also stored as vchr_destination_code.
strDepartureAirport string Optional Departure airport name or code (stored in segment extras).
strDepartureTerminal string Optional Departure terminal identifier (e.g. "T3"). Stored in segment extras.
strArrivalTerminal string Optional Arrival terminal identifier. Stored in segment extras.
strCabinClass string Optional Cabin class for this segment (e.g. "Economy", "Business"). Maps to vchr_class_of_service.
strBookingClass string Optional Booking class / RBD for this segment. Maps to vchr_class_of_booking.
strFareBasis string Optional Fare basis code for this segment (e.g. "YOWAE"). The system aggregates unique fare basis codes across all segments for the ticket.
strSegmentNo string Optional Segment sequence number within the itinerary. Stored in extras.
strAircraftIataCode string Optional IATA aircraft type code (e.g. "77W" for Boeing 777). Stored in extras.
strAircraft string Optional Aircraft name/model (e.g. "Boeing 777-300ER"). Stored in extras.
strSeatAlloted string Optional Seat number allocated to the passenger (e.g. "23A"). Stored in extras.
bintMileage number Optional Distance in miles for this segment. Used for mileage-based reporting.
strCo2Emisssion string Optional CO₂ emission value for this flight segment (for sustainability reporting). Stored in extras.
strBaggageInfo string Optional Baggage allowance information for this segment (e.g. "2PC 23KG"). Stored in extras.
chr_transaction_type string Optional Segment transaction type. Defaults to "I" (Issued) if not provided. Internal system field.

4e. GDS Remarks & Transactional Remarks

GDS Remarks — objGdsRemarks

An optional key-value object where each key is a GDS remark type/code and the value is the remark text. GDS remark codes must be configured in NutraACS. Unrecognised remark codes are rejected per ticket.

"objGdsRemarks": {
  "TOUR_CODE": "CORPRATE2024",
  "ENDORSEMENT": "NON-REFUNDABLE",
  "INVOICE_REMARK": "PROJECT-ALPHA"
}

Transactional Remarks — objTransactionalRemarks

An optional key-value object used to capture structured transactional data against configurable remark masters. Keys are the master remark name (as configured in NutraACS), and values are the detail option's code or name.

Do NOT send database primary keys. Send the human-readable master name as the key, and the detail option's vchr_code or vchr_name as the value. The system resolves to PKs internally (code is tried first, then name — both case-insensitive). 
"objTransactionalRemarks": {
  "Trip Purpose":    "Business",
  "Approval Status": "APPROVED",
  "Cost Center":     "CC-FINANCE"
}
ValidationError Code
A mandatory remark master has no value provided (and blnSkipMandatory is not set) IMPORTING_ERRORS.TRANSACTIONAL_REMARKS_MANDATORY_MISSING_{masterPk}
The master name provided does not exist in NutraACS IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_MASTER_{masterName}
The detail code/name provided is not a valid option for the given master IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_DETAIL_{masterName}

4f. Dimension Codes

Up to three optional dimension codes can be attached per ticket. Dimension codes represent analytical dimensions (e.g. cost centre, project, department) configured in NutraACS. Values are resolved by code (case-insensitive) and must be active dimension entries.

FieldTypeRequiredDescription
strDimensionCodeOne string Optional Code for the first dimension (e.g. cost centre code "CC-001"). Maps to Dimension 1 configured in the tenant.
strDimensionCodeTwo string Optional Code for the second dimension (e.g. project code "PROJ-ALPHA"). Maps to Dimension 2.
strDimensionCodeThree string Optional Code for the third dimension. Maps to Dimension 3.

5. Validations

5a. Batch-Level Validations (rejects entire request)

RuleError Response
One or more tickets in arrTickets are missing strTicketNumber { "strMessage": "KEY_MISSING_TICKET_NUMBER", "objDetails": { "arrPk": [...] } }
One or more tickets are missing strTransactionType { "strMessage": "KEY_MISSING_TRANSACTION_TYPE", "objDetails": { "arrVouchers": [...] } }
One or more tickets have a strTransactionType value other than "SALE" or "REFUND" { "strMessage": "INVALID_TRANSACTION_TYPE", "objDetails": { "arrVouchers": [...] } }

5b. Per-Ticket Validations (reported in import result)

Tickets that fail per-ticket validations are moved to the failed list and the remaining valid tickets are still processed (unless the failure is fatal).

RuleError Key
datIssue (in objDetails) is missing for a SALE transaction ISSUE_DATE_MISSING
datRefund (in objDetails) is missing for a REFUND transaction REFUND_DATE_MISSING
Supplier currency code is not found in the system IMPORTING_ERRORS.INVALID_SUPPLIER_CURRENCY
Customer currency code is not found in the system IMPORTING_ERRORS.INVALID_CUSTOMER_CURRENCY
GDS remark code in objGdsRemarks is not a recognised remark type IMPORTING_ERRORS.INVALID_GDS_REMARK_{remarkCode}
Mandatory transactional remark master has no value supplied IMPORTING_ERRORS.TRANSACTIONAL_REMARKS_MANDATORY_MISSING_{masterPk}
Transactional remark master name does not exist in the system IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_MASTER_{masterName}
Transactional remark detail value (code or name) is not a valid option for the given master IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_DETAIL_{masterName}
Dimension code in strDimensionCodeOne/Two/Three is not found IMPORTING_ERRORS.INVALID_DIMENSION_CODE_{dimensionCode}
Duplicate ticket number already exists in the system for the same customer IMPORTING_ERRORS.DUPLICATE_TICKET
Flight sector(s) in strSector or arrSegmentDetails are invalid IMPORTING_ERRORS.INVALID_SECTOR

7. Responses

Success

Returns a summary of the import result with saved, failed, and skipped ticket lists.

{
  "arrSavedTickets": ["176-2345678901"],
  "arrFailedTickets": {},
  "arrSkippedTickets": []
}

Partial Success

If some tickets pass and some fail, the successful ones are saved and failures are returned with reasons.

{
  "arrSavedTickets": ["176-2345678901"],
  "arrFailedTickets": {
    "SALE": {
      "176-9999000001": [
        "IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_MASTER_Trip Purpose"
      ]
    }
  },
  "arrSkippedTickets": []
}

Error Responses

ScenarioStatus CodeExample Response
Ticket number missing in one or more records 400 { "strMessage": "KEY_MISSING_TICKET_NUMBER", "objDetails": { "arrPk": [null] } }
Transaction type field missing 400 { "strMessage": "KEY_MISSING_TRANSACTION_TYPE", "objDetails": { "arrVouchers": [...] } }
Invalid transaction type value 400 { "strMessage": "INVALID_TRANSACTION_TYPE", "objDetails": { "arrVouchers": ["176-2345678901"] } }
Invalid or unregistered provider code 400 { "errCommon": [{ "strMessage": "INVALID_VENDOR_CODE" }] }
Unhandled server error 500 { "errCommon": [{ "strMessage": "INTERNAL_SERVER_ERROR" }] }
