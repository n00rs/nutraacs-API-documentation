NutraACS Integration Guide – Save Third Party Car Vouchers API

This API allows third-party providers (e.g. car rental platforms, GDS systems, OTA portals) to push car rental voucher transactions directly into NutraACS for both Sale and Refund processing.

1. API Endpoint

POST /api/voucher/car/save_third_party_car_vouchers

Content-Type: application/json

2. Request Structure

{
  "strProviderCode": "PROVIDERCODE",
  "objPaymentInfo": { ... },
  "arrVouchers": [
    {
      "strVoucherNo": "CV25003",
      "strTransactionType": "SALE",
      "strVoucherStatus": "I",
      ...
      "objDetails": { ... },
      "objGdsRemarks": { ... },
      "objTransactionalRemarks": { ... }
    }
  ]
}

3. Top-Level Fields

FieldTypeRequiredDescription
strProviderCode string Required The unique vendor/provider code identifying the third-party system (e.g. "NUFLIGHTS"). Must be pre-registered in NutraACS.
arrVouchers array Required Array of one or more car voucher objects to import. Each object represents a single rental transaction (Sale or Refund). See Section 4 for field details.
objPaymentInfo object Optional Payment details for invoicing the batch. When provided, all successfully saved vouchers are automatically invoiced. See Section 5.

4. Voucher Object Fields

Each element in arrVouchers represents one car rental transaction. Fields are split into root-level identifiers, Master (rental & booking info) and Details (financial data inside objDetails).

4a. Master Fields (voucher root level)

Rental company, location, passenger and booking information

FieldTypeRequiredDescription & Validation
strVoucherNo string Required Unique car voucher number (e.g. "CV25003"). Max 30 characters. Only letters, digits, / and - are allowed — special characters cause a validation error. Cannot be null or empty; the entire batch is rejected if any voucher is missing this field.
strTransactionType string Required Type of transaction. Accepted values: "SALE" or "REFUND" (case-sensitive). Any other value rejects the entire batch.
strVoucherStatus string Required Current status of the voucher. Accepted values:
  • "I" or "ISSUE" — Issued (use with SALE)
  • "R" or "REFUND" — Refunded (use with REFUND)
  • "V" or "VOID" — Voided (use with REFUND)
The status must be consistent with strTransactionType: Issue status only for SALE; Refund/Void status only for REFUND. Mismatch causes a per-voucher error.
strCarRentalCompany string Cond. Rental company name as registered in NutraACS. For third-party requests, either strCarRentalCompany (name) or strCarRentalCompanyCode (code) must be provided. If neither is sent, a validation error is raised.
strCarRentalCompanyCode string Cond. Rental company code as registered in NutraACS (e.g. "R0001"). Used for third-party resolution when strCarRentalCompany is not provided. One of the two must be present.
strCountry string Required Country name where the car is rented (e.g. "Germany"). Must match an active country configured in NutraACS. Used to determine domestic vs. international status.
strCity string Required City name where the car is rented (e.g. "Berlin"). Must match an active city in NutraACS and must belong to the provided strCountry. Mismatch causes a validation error.
strPaxName string Optional Name of the primary renter/passenger (e.g. "ARYANANDA").
strPnrNumber string Optional GDS PNR / booking reference number. Max 15 characters.
strGdsCompany string Optional GDS platform used to book the car rental. Max 20 characters. Accepted values (case-insensitive): AMADEUS, GALILEO, SABRE, WORLDSPAN, FARELOGIX, ONLINE, LCC, AIRLINE, OTHER. Defaults to "Other" if not provided. The value is title-cased before saving.
strCarConfirmNo string Optional Car rental company's confirmation number. Max 50 characters. Must be unique — duplicate confirmation numbers across the system are rejected with an error.
strSupplierConfirmNo string Optional Supplier's booking confirmation number. Max 50 characters. Must be unique — duplicate supplier confirmation numbers are rejected.
strBookingStaffId string Optional Staff code of the booking agent. Resolved to the staff account in NutraACS. If not provided for a THIRD_PARTY request, strBookingStaffEmail is used as a fallback.
strBookingStaffEmail string Optional Email of the booking agent. Used as a fallback lookup when strBookingStaffId is absent (THIRD_PARTY source only).
strBookingDetails string Optional Free-text booking details or notes. Max 500 characters.
strRentingStation string Optional Pick-up/renting station name or code (e.g. airport branch, city office). Max 100 characters.
strDropStation string Optional Drop-off station name or code. Max 100 characters.
strCategory string Optional Car category code or name (e.g. "ECONOMY", "SUV"). Max 100 characters.
strCarVehicle string Optional Vehicle make/brand name (e.g. "Citroen").
strCarModel string Optional Vehicle model name (e.g. "C5").
datStart string (date) Optional Car rental start date. Format: "DD/MM/YYYY" or "YYYY-MM-DD".
datEnd string (date) Optional Car rental end date. Same format as datStart. Must be on or after datStart.
strCustomerEmpNo string Optional Customer employee number. Used for corporate billing cross-referencing.
strAdditionalReference string Optional Additional reference text attached to the voucher (e.g. internal reference, approval code). Max 200 characters.
strTermsAndConditions string Optional Terms and conditions text to be stored against the voucher.
strRegionCode string Optional Tax region code. Used to apply the correct tax rules. If not provided, region is auto-determined from strCountry (domestic vs. international relative to the company's country).
strPickUpTerminal string Optional Airport terminal or location identifier for vehicle pick-up.
strPickUpFlightNo string Optional Incoming flight number at pick-up location (used for coordination).
strPickUpAirline string Optional Airline code/name for the incoming flight at pick-up.
datPickUp string (date) Optional Scheduled pick-up date. Format: "DD/MM/YYYY" or "YYYY-MM-DD".
strDropOffTerminal string Optional Airport terminal or location identifier for vehicle drop-off.
strDropOffFlightNo string Optional Outbound flight number at drop-off (used for coordination).
strDropOffAirline string Optional Airline code/name for the outbound flight at drop-off.
datDropOff string (date) Optional Scheduled drop-off date. Format: "DD/MM/YYYY" or "YYYY-MM-DD".

4b. Details Fields (objDetails)

Financial, rate, supplier and customer data

All amount fields should be provided as numbers or numeric strings. Currency amounts must align with their corresponding currency fields (strSupplierCurrency / strCustomerCurrency). Empty string "" is treated as zero/not provided.
FieldTypeRequiredDescription & Validation
datIssue string (date) Cond. Required when strTransactionType is "SALE". The voucher issue date. Format: "DD/MM/YYYY". Also accepted: "YYYY-MM-DD". For REFUND transactions, datRefund is used instead.
datRefund string (date) Cond. Required when strTransactionType is "REFUND". The refund processing date. Must be on or after the original issue date of the sale voucher if one exists in the system. Same date format as datIssue.
strNoOfDays string / number Optional Number of rental days (e.g. "3"). Used together with strRatePerDay for fare calculation and display.
strNoOfCars string / number Optional Number of cars rented under this voucher (e.g. "1").
strRatePerDay string / number Optional Daily rental rate per car in supplier currency. Defaults to "0" if not provided.
strTaxPerDay string / number Optional Daily tax amount per car in supplier currency. Defaults to "0" if not provided.
strBranch string Optional Branch code where the voucher is posted (e.g. "MAIN", "HO"). Must match an active branch in NutraACS. Optional for THIRD_PARTY source.
strIssuingStaffId string Optional Staff code of the issuing agent. Resolved to the staff account in NutraACS. Optional for THIRD_PARTY — falls back to strIssuingStaffEmail if absent.
strIssuingStaffEmail string Optional Email of the issuing agent. Used as a fallback lookup when strIssuingStaffId is absent (THIRD_PARTY source only).
strSupplier string Optional Supplier account code in NutraACS (the car rental supplier billed by the agency). Must be an active supplier account.
strSupplierCurrency string Optional ISO 4217 currency code for all supplier-side amounts (e.g. "USD", "AED", "INR"). Must be configured in NutraACS.
dblSupplierROE number Optional Supplier Rate of Exchange — conversion rate from supplier currency to the system base currency.
dblPublishedFare number Optional Published (rack) rental fare in supplier currency before taxes. Also accepted via dblSupplierDebit / dblSupplierCredit if this field is zero.
dblMarketFare number Optional Negotiated/market rental fare in supplier currency.
dblMarketFareCardAmount number Optional Portion of market fare charged to a corporate card.
dblTax number Optional Total tax amount on the rental in supplier currency.
dblTaxCardAmount number Optional Portion of tax charged to a card.
strTaxDetails string Optional Raw tax breakdown string as provided by the supplier. Max 250 characters.
dblSupplierCommission1 number Optional Standard commission amount from the supplier (maps to dblStdCommission).
dblSupplierCommission2 number Optional PLB (Performance Linked Bonus) commission from the supplier (maps to dblPlbCommission).
dblInputTax number Optional Input (recoverable) tax on supplier charges (e.g. GST input credit).
dblOtherInputTax number Optional Any additional input tax not covered by dblInputTax.
dblSupplierFee number Optional Processing/service fee charged by the supplier to the agency.
dblSupplierFeeTax number Optional Tax applied on the supplier fee.
dblSupplierCharge number Optional Additional supplier-imposed charge (e.g. early return fee, credit card surcharge).
dblSupplierChargeCard number Optional Portion of the supplier charge paid by card.
dblIgst number Optional Integrated GST (India — inter-state transactions).
dblSgst number Optional State GST (India — intra-state transactions).
dblCgst number Optional Central GST (India — intra-state transactions).
dblOutputIgst number Optional Output IGST charged to the customer (India — inter-state).
dblOutputSgst number Optional Output SGST charged to the customer (India — intra-state).
dblOutputCgst number Optional Output CGST charged to the customer (India — intra-state).
strCorpCardCode string Optional Corporate credit card account code. Used when the rental was paid partially or fully via a corporate card configured in NutraACS.
strMaskedCardNo string Optional Masked credit/debit card number (e.g. "1347****7854"). Do NOT send full unmasked card numbers.
strCardAuthorizationCode string Optional Payment gateway authorization code for the card transaction (e.g. "7CA79854").
strGrAccountCode string Optional Government Receivable (GR) account code for government-billed transactions.
strGrLpoNumber string Optional LPO number associated with the GR account.
strFopElement string Optional Form of Payment element from the GDS PNR (e.g. "CC VI4111111111111111/0125").
strCustomer string Optional Customer account code in NutraACS (e.g. "00001"). The voucher will be billed to this customer. Must be a valid, active customer account.
strCustomerCurrency string Optional ISO 4217 currency code for customer-facing amounts (e.g. "INR"). Defaults to the customer's configured currency if not provided.
dblCustomerROE number Optional Rate of Exchange from customer currency to the system base currency.
strCustomerLpoOrToNumber string Optional Customer's LPO or Tour Order reference number. Used for corporate billing reconciliation. Max 50 characters.
datCustomerLpoDate string (date) Optional Date of the customer LPO. Format: "DD/MM/YYYY".
dblServiceFee1 number Optional Primary service fee charged to the customer (maps to dblServiceFeeCustomer).
dblServiceFee2 number Optional Extra earning / secondary fee charged to the customer (maps to dblExtraEarningCustomer).
dblServiceFee3 number Optional Additional income / tertiary fee charged to the customer (maps to dblAdditionalIncomeCustomer).
dblDiscount1 number Optional Primary discount given to the customer (maps to dblDiscountCustomer).
dblDiscount2 number Optional PLB discount passed to the customer (maps to dblPlbDiscountCustomer).
dblDiscount3 number Optional Additional / tertiary discount for the customer (maps to dblAdditionalDiscountCustomer).
dblAgencyChargeRfd number Optional Agency-retained cancellation/refund fee charged to the customer on refund transactions.
dblCustomerOutputTax number Optional Output VAT / billing tax charged to the customer on top of the selling price.
dblExpenseOutputTax number Optional Output tax on agency expense side (maps to dblCustomerExpenseOutputTax).
dblSellingPriceCustomer number Optional Total selling price billed to the customer in customer currency (fare + taxes + fees − discounts).
strPaybackAccount1 string Optional GL account code for the first payback/rebate distribution.
dblPaybackAmount1 number Optional Amount to distribute to strPaybackAccount1.
strPaybackAccount2 string Optional GL account code for the second payback account.
dblPaybackAmount2 number Optional Amount to distribute to strPaybackAccount2.
strPaybackAccount3 string Optional GL account code for the third payback account.
dblPaybackAmount3 number Optional Amount to distribute to strPaybackAccount3.

4c. GDS Remarks & Transactional Remarks

GDS Remarks — objGdsRemarks

An optional key-value object for structured GDS remark capture. Each key is a GDS remark integration code configured in NutraACS for the CAR service type, and the value is the remark text. Unrecognised codes are rejected per voucher.

"objGdsRemarks": {
  "STRCCNO":  "123345",
  "TOUR_CODE": "CORP2024"
}

Transactional Remarks — objTransactionalRemarks

An optional key-value object for configurable transactional remark capture. The key is the remark master name as configured in NutraACS and the value is the detail option's code or name.

Do NOT send primary key IDs. Send the human-readable master name as the key and the detail's vchr_code or vchr_name as the value. The system resolves these to PKs internally — vchr_code is tried first (case-insensitive), then vchr_name as a fallback. 
"objTransactionalRemarks": {
  "TESTTI": "abc",
  "TEST":   "1234"
}
ValidationError Code
Mandatory remark master has no value supplied IMPORTING_ERRORS.TRANSACTIONAL_REMARKS_MANDATORY_MISSING_{masterPk}
Master name does not exist in NutraACS IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_MASTER_{masterName}
Detail code/name is not a valid option for the given master IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_DETAIL_{masterName}

4d. Dimension Codes

Up to three optional dimension codes can be attached per voucher. Dimension codes must match active entries configured in NutraACS. Dimensions are only applied when the corresponding dimension feature and coupon-level dimension setting are both active in the tenant configuration.

FieldTypeRequiredDescription
strDimensionCodeOne string Optional Code for Dimension 1 (e.g. cost centre "CC-001"). Values are uppercased before lookup.
strDimensionCodeTwo string Optional Code for Dimension 2 (e.g. project code "PROJ-ALPHA").
strDimensionCodeThree string Optional Code for Dimension 3.

5. Payment Info (objPaymentInfo)

When objPaymentInfo is provided, all successfully saved vouchers are automatically invoiced as a batch. If payment validation fails, no vouchers are saved and an error is returned.

FieldTypeRequiredDescription
strDocumentDate string (date) Required Invoice document date. Format: "DD/MM/YYYY".
strModeOfSale string Required Mode of sale (e.g. "Credit", "Cash").
arrPaymentOptions array Required One or more payment option objects detailing how the invoice is to be settled. See payment option fields below.

Payment Option Object (arrPaymentOptions[*])

FieldTypeRequiredDescription
intMasterCustomer string / number Required Primary key of the master customer account in NutraACS (e.g. "12566").
strPaymentType string Required Payment type (e.g. "Credit", "Cash", "Cheque", "CreditCard").
strCustomerCode string Required Customer account code being invoiced (must match the voucher's customer).
strCurrency string Required ISO 4217 currency code for the payment amount.
dblTranCurDebitedAmount number Required Total amount debited/collected in the transaction currency.
dblTranCurrRoe number Optional Rate of exchange from transaction currency to base currency. Defaults to 1 if not provided.
intPaymentOptionPk string / number Optional Payment option configuration PK. Leave empty for standard payment flows.
strCreditCardNo string Optional Credit card number (for card payment type).
strChequeNo string Optional Cheque number (for cheque payment type).
datPayment string (date) Optional Date of payment. Format: "DD/MM/YYYY".
datCheque string (date) Optional Cheque date. Format: "DD/MM/YYYY".
dblTranCurBankCharge number Optional Bank charge in transaction currency.
dblBaseCurBankCharge number Optional Bank charge converted to base currency.
objPOSDetails object Optional Point-of-sale terminal details (for card transactions).
strBatchNo string Optional Payment batch number for card reconciliation.
strCCRecoNo string Optional Corporate card reconciliation number.

6. Validations

6a. Batch-Level Validations (rejects entire request)

RuleError Response
One or more vouchers in arrVouchers are missing strVoucherNo { "strMessage": "KEY_MISSING_TICKET_NUMBER" }
One or more vouchers are missing strTransactionType { "strMessage": "KEY_MISSING_TRANSACTION_TYPE" }
strTransactionType value is not "SALE" or "REFUND" { "strMessage": "INVALID_TRANSACTION_TYPE" }
objPaymentInfo is provided but payment validation fails { "strMessage": "ERROR_IN_PAYMENT_INFORMATION", "objDetails": { "arrMessages": [...] } }
No vouchers remain to process after validation { "strMessage": "NO_RECORD_TO_SYNC" }

6b. Per-Voucher Validations (reported in import result)

Failed vouchers are moved to the error list while remaining valid vouchers are still processed.

RuleError Code
strVoucherNo contains special characters (only A-Z, 0-9, /, - allowed) IMPORTING_ERRORS.INVALID_CAR_VOUCHER_NUMBER
strVoucherNo exceeds 30 characters IMPORTING_ERRORS.VOUCHER_NO_LENGTH_EXCEEDED_30_ALLOWED
strVoucherStatus is missing IMPORTING_ERRORS.KEY_MISSING_VOUCHER_STATUS
strVoucherStatus value is not I / ISSUE / R / REFUND / V / VOID IMPORTING_ERRORS.INVALID_VOUCHER_STATUS
strVoucherStatus does not match strTransactionType (e.g. status R with type SALE) IMPORTING_ERRORS.TRANSACTION_TYPE_AND_VOUCHER_STATUS_NOT_MATCHING
Issue / refund date is missing IMPORTING_ERRORS.KEY_MISSING_ISSUE_DATE
Issue / refund date does not match format DD/MM/YYYY IMPORTING_ERRORS.INVALID_ISSUE_DATE
Refund date is earlier than the issue date of the existing sale voucher IMPORTING_ERRORS.REFUND_DATE_SHOULD_BE_GREATER_THAN_ISSUE_DATE_OF_ISSUE_COUPON
Sale voucher with the same strVoucherNo already exists in the system IMPORTING_ERRORS.CAR_VOUCHER_ALREADY_EXISTS
strCarConfirmNo already exists in the system IMPORTING_ERRORS.RENTAL_COMPANY_CONFIRM_NO_DUPLICATE
strSupplierConfirmNo already exists in the system IMPORTING_ERRORS.SUPPLIER_CONFIRM_NO_DUPLICATE
Neither strCarRentalCompany nor strCarRentalCompanyCode is provided (third-party source) IMPORTING_ERRORS.RENTAL_COMPANY_NAME_OR_CODE_REQUIRED
Rental company name/code is not found in NutraACS IMPORTING_ERRORS.RENTAL_COMPANY_DOESNOT_EXIST
strCountry is missing IMPORTING_ERRORS.KEY_MISSING_COUNTRY
strCountry is not found in NutraACS IMPORTING_ERRORS.COUNTRY_DOESNOT_EXIST
strCity is missing IMPORTING_ERRORS.KEY_MISSING_CITY
strCity is not found in NutraACS IMPORTING_ERRORS.CITY_DOESNOT_EXIST
strCity does not belong to the provided strCountry IMPORTING_ERRORS.CITY_AND_COUNTRY_NOT_MATCHING
strGdsCompany value is not a recognised GDS type IMPORTING_ERRORS.GDS_DOES_NOT_EXIST
strGdsCompany exceeds 20 characters IMPORTING_ERRORS.GDS_LENGTH_EXCEEDED_20_ALLOWED
strPnrNumber exceeds 15 characters IMPORTING_ERRORS.PNR_LENGTH_EXCEEDED_15_ALLOWED
Issuing staff code/email is not found in NutraACS IMPORTING_ERRORS.ISSUING_STAFF_DOESNOT_EXIST
Booking staff code/email is not found in NutraACS IMPORTING_ERRORS.BOOKING_STAFF_DOES_NOT_EXIST
Branch code is not found in NutraACS IMPORTING_ERRORS.BRANCH_DOES_NOT_EXIST
Supplier currency code is not recognised IMPORTING_ERRORS.INVALID_SUPPLIER_CURRENCY
Customer currency code is not recognised IMPORTING_ERRORS.INVALID_CUSTOMER_CURRENCY
GDS remark integration code in objGdsRemarks is not configured for the CAR service IMPORTING_ERRORS.INVALID_GDS_REMARK_{remarkCode}
Mandatory transactional remark master has no value provided IMPORTING_ERRORS.TRANSACTIONAL_REMARKS_MANDATORY_MISSING_{masterPk}
Transactional remark master name not found in NutraACS IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_MASTER_{masterName}
Transactional remark detail code/name is not a valid option IMPORTING_ERRORS.INVALID_TRANSACTIONAL_REMARK_DETAIL_{masterName}
Dimension code not found in NutraACS IMPORTING_ERRORS.INVALID_DIMENSION_CODE_{dimensionCode}
Supplier-side fields provided but service is configured as own-service (no external supplier) IMPORTING_ERRORS.OWN_SERVICE_VOCHER

8. Responses

Success

{
  "Success": {
    "Sale":   ["CV25003"],
    "Refund": []
  },
  "Error": {
    "Sale":   {},
    "Refund": {}
  },
  "arrPaymentErrors": [],
  "strDocumentNumber": "INV-2026-00123"
}
strDocumentNumber is only present when objPaymentInfo was provided and the invoice was successfully generated.

Partial Success

Valid vouchers are saved; failed ones are listed under Error with reasons.

{
  "Success": {
    "Sale":   ["CV25003"],
    "Refund": []
  },
  "Error": {
    "Sale": {
      "CV25999": [
        "IMPORTING_ERRORS.RENTAL_COMPANY_DOESNOT_EXIST",
        "IMPORTING_ERRORS.INVALID_SUPPLIER_CURRENCY"
      ]
    },
    "Refund": {}
  },
  "arrPaymentErrors": []
}

Error Responses

ScenarioStatus CodeExample Response
Voucher number missing in one or more records 400 { "strMessage": "KEY_MISSING_TICKET_NUMBER" }
Transaction type field missing 400 { "strMessage": "KEY_MISSING_TRANSACTION_TYPE" }
Invalid transaction type value 400 { "strMessage": "INVALID_TRANSACTION_TYPE" }
Payment information is invalid 400 { "strMessage": "ERROR_IN_PAYMENT_INFORMATION", "objDetails": { "arrMessages": ["INVALID_CUSTOMER_ACCOUNT"] } }
Invalid or unregistered provider code 400 { "errCommon": [{ "strMessage": "INVALID_VENDOR_CODE" }] }

Unhandled server error 500 { "errCommon": [{ "strMessage": "INTERNAL_SERVER_ERROR" }] }
